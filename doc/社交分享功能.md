# 新增功能需求

本次新增两个功能模块：**三工具关联机制** 和 **社交分享卡片**。

---

## 1. 三工具关联机制

### 背景

当前 Angel Number、Gematria、Numerology 三个工具彼此孤立。用户用完一个工具后容易直接离站，而竞品均没有跨工具联动。打通三者可以显著提升停留时间、页面浏览数和 SEO 内链权重。

### 设计原则

- **并列不合并** — 三工具保持独立入口，不强制整合
- **数值命中才触发** — 仅在计算结果命中有意义的数字时出现推荐，不是每次都推
- **标注来源** — 每条关联推荐必须明确标注体系来源，不混淆
- **轻量出口** — 每页最多 1-2 个跨工具推荐入口，样式克制，用户可忽略

### 关联路径

#### 路径 A：Numerology → Angel Number（优先实现）

**触发：** 始终触发，每次 Numerology 计算都有结果数字。

**实现：** 在 Numerology 结果页的每个数字（Life Path / Expression / Soul Urge / Personality / Birthday）旁，添加跳转链接。

**文案示例：**
```
Life Path Number: 7
7 also carries meaning as an Angel Number →   ← 链接到 /angel-number/7/
```

**注意：** 链接文案用"also carries meaning as an Angel Number"而非"你的 Angel Number 是 7"——两个体系语境不同，不能直接等同。Numerology 描述的是人格属性，Angel Number 描述的是当下讯息。

---

#### 路径 B：Gematria → Angel Number

**触发：** 任意 Gematria 系统的计算结果命中以下数字时才出现：

`111 / 222 / 333 / 444 / 555 / 666 / 777 / 888 / 999 / 1111 / 1212 / 1234`

**实现：** 在命中的计算结果旁显示一个可折叠卡片，标注 `Spiritual Interpretation`，内容为该数字的含义标题 + 2 句解读 + 链接。默认折叠，不强制展开。

**示例：** 用户输入 "Jesus"，English Gematria = 444，出现：

```
┌─────────────────────────────────────────┐
│ [Spiritual Interpretation]              │
│ 444 — Protection & Stability            │
│ Your angels are surrounding you...      │
│ → Read full Angel Number 444 meaning   │
└─────────────────────────────────────────┘
```

---

#### 路径 C：Angel Number → Gematria

**触发：** 始终触发。

**实现：** 在每个 Angel Number 详情页底部增加一个区块：

```
Curious which words equal 444 in Gematria?
→ Try the Gematria Calculator
```

链接跳转到 `/calculator/gematria/`，并预填 `?value=444` 作为查询参数。

---

#### 路径 D：Gematria → Numerology

暂不实现。用户在计算文字 Gematria 时对个人 Numerology 画像通常没有需求，路径不自然。

---

### 共享数字含义数据库

三工具需要共用一份底层数字象征库（`/data/number-base-meanings.json`），避免同一个数字在不同工具中出现描述矛盾。

数据结构：

```json
{
  "7": {
    "keywords": ["灵性", "内省", "智慧"],
    "numerology_desc": "你天生是探求真理的人，内省与智慧是你的核心特质",
    "angel_desc": "你正走在灵性觉醒的路上，相信你的直觉"
  }
}
```

- `numerology_desc` — 用于 Numerology Calculator 结果页（人格属性语境）
- `angel_desc` — 用于 Angel Number 详情页（当下讯息语境）
- 两套文案共享关键词，但分开维护，不互用

## 2. 社交分享卡片

### 背景

目标用户（18-35 岁女性灵性探索者）高度活跃于 Instagram 和 TikTok。用户分享一张写着"我的 Angel Number 是 444"的精美卡片，本质上是免费的品牌曝光。当前三个工具均没有分享功能。

### 覆盖范围

三个工具均支持：Angel Number 详情页、Gematria 计算结果页、Numerology 结果页。

### 卡片内容

每张卡片包含：

- 来源工具标注（如 `ANGEL NUMBER` / `LIFE PATH NUMBER` / `GEMATRIA`）
- 大号数字
- 含义标题
- 2-3 句解读文案
- 网站域名

卡片内容对没用过产品的人也能看懂，起到引流作用。

### 三工具卡片配色

| 工具 | 背景 | 数字颜色 | 调性 |
|------|------|---------|------|
| Angel Number | 深紫 → 靛蓝渐变 | 紫粉 #f0abfc | 星空神秘 |
| Numerology | 深蓝 → 靛色渐变 | 天蓝 #7dd3fc | 深邃沉静 |
| Gematria | 深金棕 → 暗金渐变 | 金黄 #fcd34d | 古典厚重 |

三种风格独立，用户分享出去的卡片自带工具识别度。

### 分享按钮层级

**第一层 — 下载图片（最显眼，主按钮）**

Instagram 和 TikTok 没有网页直接分享接口，下载后自行上传是主流路径。放最显眼位置，紫色实心大按钮。

**第二层 — 平台直达（三个并排中等按钮）**

- X (Twitter) — 直接拼 `intent/tweet` URL，无需 API Key
- Instagram — 跳转后提示用户下载上传
- TikTok — 跳转后提示用户下载上传

**第三层 — 复制链接（小号线框按钮，兜底）**

复制当前结果页 URL，显示 Toast 提示"链接已复制"。

### 技术实现

**卡片图片生成（推荐方案）：**

复用现有 `/api/og` 服务端渲染管线，增加 `tool` 参数区分三种工具样式：

```
/api/og?tool=angel&number=444
/api/og?tool=numerology&number=7&label=Life+Path
/api/og?tool=gematria&number=74&word=Jesus
```

**备选方案：** 前端 `html2canvas` 截图，零服务器成本，快速上线，但深色背景偶有兼容问题。

**Twitter 分享拼接示例：**

```
https://twitter.com/intent/tweet?text=My angel number is 444 — Protection %26 Divine Guidance ✦ Discover yours at [domain]
```

**复制链接：**

```javascript
navigator.clipboard.writeText(window.location.href)
```
